# Research
Предположительно у нас должна быть микро сервисная архитектура, т.к. наверняка наш проект будет вклиниваться в уже существующую систему.  

## Ключевые принципы

* Нельзя допускать ошибок первого рода (ложно положительный результат); 
* Каждая рекомендация должна четко ссылаться на существующий документ без потери точности информации, следовательно, каждое утверждение должно быть подкреплено данными (клиническими рекомендациями или предоставляемыми документами);
* При отсутствии заболевания выдавать советующие сообщение об ошибке (см предыдущий пункт);
* Чтобы четко отвечать на вопросы пользователя данные должны быть предобработаны, размечены;
* Один и тот же запрос должен давать один и тот же ответ;
* Возможность простого удаление или добавления новых источников.

Для Backend разработки будет использоваться язык программирования python и его библиотеки. RESTful API

## Взаимодействие API

Для взаимодействия между fronted и backend можно воспользоваться WebSocket / Server-Sent Events (SSE). ML модель может изначально загружаться в память при старте FastAPI (но не подойдет для больших моделей), либо сделать отдельный микросервис, к которому будут обращаться. Можно кэшировать часто повторяющиеся текста.

Для удобства пользователя было бы неплохо чтобы было исправление опечаток, например PySpellChecker или Pymorphy2 (но он может не знать редких слов). Приведение к нижнему регистру. Также из текста запроса необходимо извлекать ключевые слова, медицинские термины и классифицировать намерения (лечение, диагностика, профилактика). Дальше поиск соответствующих фрагментов текста необходимых для ответа на запрос. Формирование ответа и передача его пользователю (можно передавать JSON файлы). Перед отправкой надо проверять правильно ли указан документ и не сгенерировался ли он.

## Работа с документами

Документарная база может располагаться в PostgreSQL для быстрого поиска по ключевым словам. Данные должны быть векторизированы (название книги, название глав, теги внутри текста) и храниться в другой базе данных.

Необходим отдельный скрипт для администрирования документов. При добавлении необходимо делать векторизацию и добавить новую информацию в базу данных. При удалении надо убрать все связанные записи из документальной базы. Прямым взаимодействием с документами должен заниматься только администратор.
